**Интересный факт:** создавал новый экземпляр Warrior после каждого нанесенного урона в самом же Warrior в методе hit, но не учел, что этот метод будут переиспользовать потомки.

4.2.

полиморфизм подтипов - позволяет реализовывать в том числе перегрузки методов, т.к. можно вызвать хоть и единожды определенный, но вызываемый с разным набором аргументов метод.

параметрический полиморфизм - позволяет писать код опираясь на общий тип, то есть не погружаясь в детали реализации.
Подобно тому, как хорошим приемом для снижения ошибок может быть написание более полиморфного, а не мономорфного кода. Т.к. в первом случае у нас меньше свободы в том, какие методы мы сможем вызывать.

```python
class Animal:
    def foo(self):
        pass

class Cat(Animal):
    def foo(self):
        print("Кошка мурлычет")

class Bird(Animal):
    def foo(self):
        print("Птица поет")

def animals_foo():
    barsik = Cat()
    galka = Bird()
    animals = [barsik, galka]
    check_list(animals)

def check_list(animals: list[Animal]) -> None:
    # отчищаю список
    animals.clear()
    for i in range(500):
        if i%2==0:
            animals.append(Cat())
        else:
            animals.append(Bird())
    for animal in animals:
        animal.foo()

animals_foo()
```

Вывод
```
Птица поет
Кошка мурлычет
Птица поет
Кошка мурлыч...
```


В python утиная типизация и по полиморфизм подтипов, будет вызываться метод объекта от класса, который реально находится по ссылке.
